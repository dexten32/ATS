Project Overview: ATS Pro - AI-Powered Resume Analysis

1. Problem It's Solving
Manual resume screening is time-consuming and prone to human bias or error. Recruiters often struggle to quickly identify the most relevant candidates from a large pool of applicants. ATS Pro solves this by automating the initial screening process using Natural Language Processing (NLP) and semantic matching to objectively score resumes against specific job descriptions.

2. Impact
- Efficiency: Drastically reduces the time spent on manual resume screening.
- Accuracy: Improves the quality of candidate shortlisting through objective, data-driven semantic matching.
- Insights: Provides actionable feedback on candidate maturity, technical depth, and specific skill gaps, empowering recruiters to make informed decisions quickly.

3. Scope
The application encompasses:
- File ingestion and validation for common resume formats (PDF, DOCX, TXT).
- Text extraction and sanitization.
- AI-based parsing to extract skills, experience, and contact information.
- Semantic scoring (TF-IDF and domain-specific heuristics) against job descriptions.
- Secure, persistent storage of resumes, job descriptions, and analysis results.
- A modern, responsive web interface for users to interact with the system.

4. Database Entities
The system uses SQLite with SQLAlchemy ORM and defines three main entities:
- Resume: Stores candidate details including `filename`, `file_path`, `content` (raw text), `candidate_name`, `email`, `phone`, `skills` (JSON), `experience_years`, and `uploaded_at`.
- JobDescription: Stores `title`, `raw_text`, `extracted_features` (JSON for required skills, min experience, etc.), and `created_at`.
- AnalysisResult: Links a Resume and JobDescription. Stores `overall_score`, `skill_score`, `experience_score`, `confidence_level`, `feedback` (JSON containing missing skills, suggestions, strengths), `metadata_info`, and `created_at`.

5. Application Flow
- User interacts with the web interface to upload a resume file and paste a job description.
- The frontend sends a multipart/form-data POST request to the backend API (`/api/v1/analyze`).
- Backend (FastAPI) ingests the file, validating and saving it temporarily.
- Text is extracted and sanitized from the resume file.
- Parsing services extract candidate details (skills, experience, contact info) and save the Resume to the database.
- The job description is parsed for required features and saved to the database.
- A scoring service compares the resume data against the job description features.
- A feedback service generates actionable insights based on the match results.
- The analysis result is saved to the database and returned to the frontend as a JSON response.
- The frontend updates the UI to display the scores and feedback.

6. Data Flow
- Input: Resume File (Binary) & Job Description (Text) -> Frontend.
- Transfer: FormData -> Backend API.
- Processing:
  - File -> Extracted Text -> Parsed Features (Skills, Exp) -> Database (resumes table).
  - JD Text -> Parsed Features (Required Skills) -> Database (job_descriptions table).
  - Resume Features + JD Features -> Scoring Engine -> Scores & Feedback -> Database (analysis_results table).
- Output: JSON Response -> Frontend State -> Rendered UI Components.

7. Frontend Flow
- Built with React (Vite) and styled with Tailwind CSS / vanilla CSS.
- Initial State: App renders the main landing page with a `ResumeUploader` component and a professional header containing a `ThemeSwitcher`.
- Action: User fills in the form and submits. `App.jsx` sets `isLoading` to true.
- API Call: `handleAnalyze` function makes an async `fetch` request to the backend.
- Result Handling: 
  - On success, the response JSON is saved to the `results` state, which triggers the rendering of the `AnalysisResults` component to display the match score and feedback.
  - On failure, an error message is displayed.
- The UI features a glassmorphism/premium design that supports multiple themes (Light, Ocean, Dark, Midnight).

8. MVP (Minimum Viable Product)
The core MVP functionality includes:
- A functional web UI allowing users to upload a single resume and provide a job description text.
- Backend API capable of extracting text from the uploaded file.
- Basic NLP/keyword matching to calculate a match score based on skills and experience.
- Display of the overall match score, matched skills, missing skills, and basic feedback to the user on the frontend.
- Persistent database storage to keep a history of uploaded resumes and their corresponding analysis results.
